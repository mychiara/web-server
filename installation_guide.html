Panduan Instalasi Server & Dashboard

Ubah Laptop bekas atau Mini PC Anda menjadi pusat server lokal yang handal. Langkah pengerjaan lengkap, terperinci, tanpa satu pun bagian yang terlewat.

Pengenalan

MasanDigital Dashboard dirancang untuk berjalan di atas sistem berbasis Docker pada platform Linux. Dibandingkan menggunakan virtual machine (VM) yang boros memori, server lokal berbasis Linux Server harian mengonsumsi sumber daya perangkat keras yang sangat minimal, sangat cocok dipasang di perangkat Mini PC hemat daya atau Laptop dengan layar retak.

Linux OS

Menggunakan sistem operasi Ubuntu Server Headless untuk efisiensi RAM.

SSH Secure Access

Mengontrol jarak jauh dari Laptop/PC utama tanpa perlu keyboard & layar tambahan.

Docker Containerized

Menjalankan database dan server aplikasi secara terisolasi dan mandiri.

Interactive Web GUI

Panel visual elegan untuk memantau performa, LXD, dan port jaringan secara dinamis.

1. Instalasi OS Linux (Ubuntu Server)

Tahap pertama adalah mengganti OS Laptop/Mini PC target menjadi Ubuntu Server. Proses ini akan menghapus seluruh data di disk internal perangkat tersebut agar performa optimal.

Unduh Berkas & Alat Bantu

Klik tombol berikut untuk mendownload software resmi di PC utama Anda:

Download Ubuntu Server ISO Download Rufus Utility
1.1

Membuat Media Bootable Flashdisk

Tancapkan flashdisk kosong (kapasitas minimal 8 GB) pada PC utama Anda, lalu lakukan prosedur ini:

  1. Buka aplikasi Rufus yang telah diunduh.
  2. Pada menu drop-down Device, pilih Flashdisk Anda.
  3. Pada Boot Selection, klik tombol SELECT lalu arahkan ke file ISO Ubuntu Server.
  4. Pada bagian Partition Scheme, pilih GPT (bila Mini PC modern UEFI) atau MBR (bila Laptop jadul).
  5. Tekan tombol START di bagian bawah.
  6. Jika ada kotak dialog konfirmasi, klik OK. Tunggu 5-10 menit sampai status menjadi READY. Cabut flashdisk.
1.2

Booting Melalui Flashdisk di Perangkat Server

Tancapkan flashdisk ke port USB Mini PC atau Laptop target, hubungkan keyboard dan monitor, nyalakan mesin, dan tekan tombol masuk Boot Menu secara terus-menerus:

  • Intel NUC, ASUS: Tekan tombol F8 atau F12
  • Lenovo, HP: Tekan tombol F12 atau F9
  • Acer, Dell: Tekan tombol F12
  • Beelink, Mini PC Cina: Tekan tombol F11 atau Esc

Setelah menu boot tampil di layar monitor, pilih item yang sesuai dengan merk Flashdisk USB Anda.

1.3

Panduan Langkah Demi Langkah Pengaturan OS Ubuntu

Setelah masuk ke halaman installer hitam Ubuntu, lakukan pengaturan berikut menggunakan tombol arah keyboard dan Enter:

  1. Language: Pilih English.
  2. Keyboard Configuration: Pilih English (US), tekan Enter pada Done.
  3. Type of Install: Pilih Ubuntu Server.
  4. Network Connections: Pasang kabel LAN (Ethernet). Sistem akan secara otomatis mendeteksi konfigurasi DHCP router Anda dan menampilkan IP Address server (contoh: 192.168.1.15). Catat alamat IP ini!
  5. Proxy & Archive Mirror: Kosongkan dan langsung pilih Done.
  6. Storage Configuration: Pilih opsi Use an entire disk. Centang kotak "Set up this disk as an LVM group". Pilih disk target penyimpanan internal Anda, lalu tekan Done. Konfirmasi penghapusan data dengan memilih Continue.
  7. Profile Setup: Masukkan detail profil administrator server:
    • Your Name: masandigital
    • Your Server's Name: server-masandigital
    • Pick a Username: masandigital
    • Password: isi dengan sandi login server Anda
  8. SSH Setup: Gunakan tombol spasi untuk mencentang "[X] Install OpenSSH server". Hal ini krusial agar Anda bisa meremote server nantinya.
  9. Featured Server Snaps: Langsung arahkan ke Done (tanpa perlu mencentang apapun).
  10. Tunggu & Reboot: Tunggu hingga proses instalasi menampilkan teks "Install complete!". Arahkan kursor ke tombol Reboot Now lalu tekan Enter. Cabut flashdisk ketika diminta.

2. Hubungkan ke Server via SSH

Kini server Anda sudah aktif dan monitor fisiknya bisa dicabut. Semua aktivitas penginstalan aplikasi selanjutnya dilakukan secara remote dari komputer utama Anda.

Unduh Alat Remote (Bila Diperlukan)

Jika menggunakan sistem operasi Windows, Anda bisa mendownload terminal client visual:

Download PuTTY Client
2.1

Koneksi Menggunakan Terminal / PowerShell

Buka program PowerShell di Windows atau Terminal di macOS/Linux PC utama Anda, lalu ketik perintah berikut:

PowerShell / Terminal Host 
ssh masandigital@IP_SERVER_LINUX_ANDA
# Contoh: ssh masandigital@192.168.1.15

Ketik yes saat pertama kali muncul validasi fingerprint sidik jari server, lalu ketik password server Anda (password tidak akan memunculkan tanda bintang/ketikan saat diinput, langsung ketik saja lalu tekan Enter).

2.2

Koneksi Menggunakan Aplikasi PuTTY

  1. Buka aplikasi PuTTY.
  2. Ketikkan IP address server pada kolom Host Name (or IP address).
  3. Pastikan port bernilai 22 dan Connection Type adalah SSH.
  4. Klik tombol Open.
  5. Gunakan username masandigital dan masukkan kata sandi server Anda.
2.3

Mengonfigurasi IP Static (IP Tetap) Server

Secara default, modem IndiHome/IndiBiz akan mengacak IP Address laptop server Anda setiap kali modem dinyalakan ulang. Agar IP Address server Anda tidak berubah-ubah, sangat disarankan menyetel IP Static menggunakan salah satu metode berikut:

Metode A: Menggunakan DHCP Binding di Modem (Sangat Direkomendasikan & Paling Aman)

Metode ini merekatkan MAC Address kartu LAN laptop ke IP tertentu secara permanen di sisi router modem IndiHome Anda. Tidak ada risiko pemutusan koneksi SSH secara permanen.

  1. Ketik perintah ini di terminal SSH untuk melihat MAC Address kabel LAN server Anda:
    Terminal SSH 
    ip link show
    Catat baris kode heksadesimal setelah kata link/ether (contoh: 00:1a:2b:3c:4d:5e) pada interface ethernet Anda (biasanya bernama eth0 atau enp2s0).
  2. Buka halaman admin modem IndiHome Anda di web browser (biasanya alamat IP di http://192.168.1.1, http://192.168.18.1, atau http://192.168.100.1).
  3. Login dengan kredensial modem Anda (lihat stiker di belakang modem).
  4. Masuk ke menu Application / Network -> **DHCP Server** -> **DHCP Binding** / **IP & MAC Binding** / **Static Lease**.
  5. Klik Add New, masukkan MAC Address server Anda, lalu tentukan IP Address tetap yang Anda kehendaki (misal: 192.168.1.15). Simpan perubahan dan restart modem.
Metode B: Mengubah Konfigurasi Netplan di Linux Server

Metode ini mengunci IP langsung dari konfigurasi internal Linux Server Anda.

  1. Buka berkas konfigurasi jaringan Netplan:
    Terminal SSH 
    sudo nano /etc/netplan/00-installer-config.yaml
  2. Ubah isinya menjadi seperti di bawah ini (sesuaikan interface enp2s0 dengan nama interface ethernet Anda, dan 192.168.1.1 dengan IP gateway modem Anda):
    Berkas YAML Netplan 
    network:
  version: 2
  renderer: networkd
  ethernets:
    enp2s0:
      dhcp4: no
      addresses:
        - 192.168.1.15/24
      nameservers:
        addresses: [8.8.8.8, 1.1.1.1]
      routes:
        - to: default
          via: 192.168.1.1
  3. Simpan perubahan (Ctrl+O, Enter, Ctrl+X), kemudian terapkan konfigurasi jaringan baru:
    Terminal SSH 
    sudo netplan apply

3. Instalasi Docker & Docker Compose

Ikuti petunjuk perintah (command prompt) di bawah ini secara persis. Jalankan baris demi baris perintah di dalam jendela koneksi SSH terminal server Anda:

3.1

Memperbarui Paket Repositori Sistem

Terminal SSH 
sudo apt-get update && sudo apt-get upgrade -y
3.2

Memasang Dependensi Prasyarat

Terminal SSH 
sudo apt-get install -y ca-certificates curl gnupg lsb-release
3.3

Mendaftarkan GPG Key Resmi Docker

Terminal SSH 
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
3.4

Mengunduh & Mendaftarkan Repositori Paket Docker

Terminal SSH 
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
3.5

Memasang Engine Docker & Ekstensi Compose Terbaru

Terminal SSH 
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
3.6

Konfigurasi Hak Akses User Non-Sudo (Opsional)

Agar user Anda tidak perlu memanggil awalan sudo setiap memanggil engine Docker:

Terminal SSH 
sudo usermod -aG docker $USER
newgrp docker
3.7

Memvalidasi Operasional Docker

Terminal SSH 
docker --version
docker compose version

4. Kloning & Deploy MasanDigital Dashboard

Instalasi software dasar sudah selesai. Sekarang kita tinggal mendownload source-code dashboard dan mengeksekusi script instalasi otomatis.

4.1

Download Source Code Dari GitHub

Masuk ke folder utama server Anda, lalu lakukan kloning repositori:

Terminal SSH 
cd ~
git clone https://github.com/mychiara/web-server.git masandigital_dashboard
4.2

Jalankan Script Instalasi Otomatis

Masuk ke dalam direktori hasil kloning, berikan hak eksekusi pada script, lalu jalankan script pemasang otomatis:

Terminal SSH 
cd masandigital_dashboard
chmod +x install.sh
sudo bash install.sh
Catatan Proses Sukses

Setelah server mendownload image dan menyusun container Docker, terminal akan menampilkan konfirmasi sukses: "INSTALASI SUKSES! Silakan akses Dashboard Admin di: http://<IP_SERVER>".

5. Konfigurasi Awal (Setup Wizard)

Setelah instalasi sukses dideploy, ikuti langkah-langkah visual berikut di perangkat PC utama atau Smartphone Anda:

  1. Buka aplikasi web browser (Chrome/Edge/Safari/Firefox).
  2. Ketikkan IP address server Anda pada address bar (Contoh: http://192.168.1.15) lalu tekan Enter.
  3. Layar browser Anda akan memuat halaman Setup Wizard - MasanDigital Dashboard.
  4. Ketikkan password administratif yang ingin Anda buat, lalu konfirmasi ulang password tersebut pada kolom di bawahnya.
  5. Simpan password Anda. Halaman web otomatis dialihkan ke form login utama.
  6. Masukkan password admin yang baru saja Anda buat untuk mulai masuk ke sistem dashboard server.

6. Akses Publik (Cloudflare Tunnel)

Cloudflare Tunnel memungkinkan Anda menghubungkan server lokal ke internet secara aman menggunakan nama domain Anda sendiri (misal: panel.masandigital.com) secara gratis, tanpa perlu melakukan port forwarding dan melewati CGNAT modem IndiHome.

6.1

Prasyarat Awal

  • Miliki akun **Cloudflare** gratis di dash.cloudflare.com.
  • Miliki domain aktif yang sudah dihubungkan ke Cloudflare (Nameservers domain diarahkan ke Cloudflare).
6.2

Membuat Tunnel Baru di Cloudflare Zero Trust

  1. Masuk ke dasbor **Cloudflare Zero Trust** di one.dash.cloudflare.com.
  2. Buka menu **Networks** -> **Tunnels** di sidebar kiri, lalu klik tombol **Create a Tunnel**.
  3. Pilih tipe **Cloudflared** (default) dan klik **Next**.
  4. Beri nama tunnel Anda (contoh: server-rumah) lalu klik **Save Tunnel**.
6.3

Memasang Daemon Cloudflared di Laptop Server

Di halaman Cloudflare, pilih tab sistem operasi **Debian** dan arsitektur **amd64** (karena Ubuntu Server kita berjalan di arsitektur 64-bit). Cloudflare akan menampilkan satu perintah instalasi panjang yang berisi token unik Anda.

Salin perintah tersebut dan jalankan di terminal SSH server Anda. Perintah tersebut akan terlihat mirip seperti ini:

Terminal SSH 
curl -L --output cloudflared.deb https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb && \
sudo dpkg -i cloudflared.deb && \
sudo cloudflared service install <TOKEN_UNIK_AKUN_ANDA>

Setelah dijalankan, status koneksi tunnel di dasbor Cloudflare web akan berubah menjadi hijau (**HEALTHY / ACTIVE**). Klik tombol **Next** di dasbor web Cloudflare.

6.4

Mengarahkan Domain ke Dashboard Lokal

Di halaman **Route Traffic (Public Hostname)** di dasbor Cloudflare:

  1. Subdomain: Masukkan subdomain yang diinginkan (misal: panel atau dashboard).
  2. Domain: Pilih nama domain Anda yang terdaftar di Cloudflare (misal: masandigital.com).
  3. Service Type: Pilih **HTTP**.
  4. URL: Masukkan localhost:80 (karena dashboard utama berjalan di port 80).
  5. Klik **Save Tunnel** di bagian bawah.

Selamat! Sekarang Anda dapat mengakses MasanDigital Dashboard Anda dari manapun di seluruh dunia menggunakan koneksi aman HTTPS di URL domain Anda: https://panel.domainanda.com.

7. Operasional & Perintah Dasar Harian

Simpan atau catat instruksi di bawah ini untuk kebutuhan pengelolaan/pemeliharaan server di masa depan:

Mematikan & Menyalakan Dashboard

Terminal SSH 
# Navigasi ke folder proyek
cd ~/masandigital_dashboard

# Menghentikan layanan container dashboard
docker compose down

# Menyalakan kembali kontainer di latar belakang
docker compose up -d

# Melihat output logs program backend dashboard
docker compose logs -f

Konfigurasi Penutup Layar Laptop (Bila Menggunakan Laptop)

Bila server Anda berupa laptop, OS Ubuntu Server akan sleep saat layar ditutup. Agar laptop tetap menyala normal saat ditutup, lakukan perintah ini:

Terminal SSH 
sudo nano /etc/systemd/logind.conf

Gunakan tombol navigasi keyboard untuk mencari baris konfigurasi berikut. Hilangkan tanda pagar # di awal baris dan ubah valuenya menjadi ignore:

HandleLidSwitch=ignore
HandleLidSwitchExternalPower=ignore
HandleLidSwitchDocked=ignore

Simpan perubahan berkas dengan menekan tombol kombinasi Ctrl + O, tekan Enter, lalu tekan Ctrl + X untuk keluar. Terapkan konfigurasi dengan merestart daemon:

Terminal SSH 
sudo systemctl restart systemd-logind

FAQ & Solusi Kendala

Berikut adalah daftar pemecahan masalah (FAQ) seputar instalasi server:

Pastikan IP address target yang diinput di browser sudah benar. Selain itu, pastikan port 80 diperbolehkan oleh firewall sistem operasi Linux Server Anda:

sudo ufw allow 80/tcp
sudo ufw reload

Anda cukup menghapus file proteksi keamanan di folder data dashboard dan merestart kontainer untuk memulai ulang Setup Wizard:

cd ~/masandigital_dashboard
sudo rm data/security_config.json
docker compose restart

Karena Anda mengunduh dashboard menggunakan Git, Anda dapat memperbarui kode lokal langsung dengan menarik update repository GitHub dan membuild ulang image Docker Anda:

cd ~/masandigital_dashboard
git pull origin main
docker compose up -d --build

Error ini berarti port 80 pada server Anda sedang dipakai oleh program server web lokal lain (seperti Apache2 atau Nginx bawaan Linux host). Anda perlu menonaktifkannya sebelum menjalankan Docker:

sudo systemctl stop apache2
sudo systemctl disable apache2
# atau untuk nginx bawaan host:
sudo systemctl stop nginx
sudo systemctl disable nginx
Kode berhasil disalin!
