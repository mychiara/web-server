Panduan Pemasangan Server & Dashboard

Panduan komprehensif langkah-demi-langkah bagi pemilik Laptop / Mini PC untuk menginstal OS Linux, Docker, Docker Compose, hingga aplikasi MasanDigital Dashboard siap digunakan.

Pengenalan

Dokumentasi ini ditulis khusus untuk membantu Anda menyiapkan infrastruktur server mandiri menggunakan perangkat fisik Laptop atau Mini PC yang menganggur. Melalui panduan ini, perangkat Anda akan disulap menjadi sistem server lokal/publik yang handal dengan panel pengelolaan web yang modern.

Konsep Infrastruktur

Mini PC/Laptop Anda akan berjalan menggunakan sistem operasi Linux Server (Headless/tanpa GUI) untuk memaksimalkan efisiensi RAM dan CPU. Seluruh komponen (Database, Web Server, Monitoring) dibungkus menggunakan Docker agar sistem terisolasi dan mudah dikelola dari perangkat lain di jaringan rumah Anda.

1. Instalasi OS Linux (Ubuntu Server)

Langkah awal adalah menginstall sistem operasi Linux pada perangkat Laptop atau Mini PC Anda. Kami merekomendasikan **Ubuntu Server 24.04 LTS / 22.04 LTS** karena stabilitasnya.

Download Bahan & Tool Resmi

Unduh seluruh berkas resmi di bawah ini langsung dari PC/Laptop utama Anda:

Download Ubuntu Server ISO Download Rufus (Tool Flash USB)

Tahap 1.1: Flash ISO ke Flashdisk

  1. Tancapkan Flashdisk kosong (minimal kapasitas 8 GB) ke PC/Laptop utama Anda.
  2. Jalankan program Rufus yang telah diunduh.
  3. Pada kolom Device, pilih Flashdisk Anda.
  4. Pada kolom Boot selection, klik SELECT dan pilih file ISO Ubuntu Server yang sudah diunduh.
  5. Di bagian Partition scheme, pilih GPT (bila Mini PC Anda mendukung UEFI modern) atau pilih MBR (bila laptop lama).
  6. Klik tombol START. Jika muncul peringatan, pilih "Write in ISO Image mode" dan klik OK. Tunggu hingga status bertuliskan READY.

Tahap 1.2: Booting ke Installer di Laptop/Mini PC

  1. Colok Flashdisk installer yang sudah jadi ke port USB Laptop/Mini PC target.
  2. Nyalakan Laptop/Mini PC, lalu tekan tombol menu boot berulang-ulang hingga muncul layar pilihan boot. Tombol boot bervariasi sesuai merk perangkat:
    • ASUS / Intel NUC: F8 atau F12
    • Lenovo / HP: F12 atau F9
    • Acer / Dell: F12
    • Motherboard Lain / Beelink: F11 atau Esc
  3. Pilih opsi boot yang merujuk pada nama USB Flashdisk Anda (misal: UEFI: SanDisk, Partition 1).

Tahap 1.3: Proses Pemasangan Ubuntu OS

Ikuti konfigurasi instalasi pada monitor Laptop/Mini PC Anda sebagai berikut:

1

Bahasa & Layout Keyboard

Pilih bahasa English dan layout keyboard default US.

2

Type of Installation

Pilih Ubuntu Server (pilihan standar, bukan minimal).

3

Network Connections (Koneksi Jaringan)

Direkomendasikan menggunakan kabel LAN agar koneksi stabil. Installer akan menampilkan IP DHCP yang didapatkan server (misal: 192.168.1.15). Catat IP ini!

4

Storage Configuration (Partisi Disk)

Pilih opsi Use an entire disk. Centang bagian "Set up this disk as an LVM group". Pilih harddisk internal Laptop/Mini PC Anda. Pilih Done dan pilih Continue saat ditanyakan konfirmasi format.

5

Profile Setup (Akun Pengguna)

Isi informasi akun server Anda:

  • Your name: masandigital
  • Your server's name: server-masandigital
  • Pick a username: masandigital
  • Password: buat sandi yang kuat dan mudah diingat

6

SSH Setup (Akses Remote)

Pada layar pilihan SSH, tekan Spasi untuk mencentang "[X] Install OpenSSH server". Ini wajib agar kita dapat mengontrol server ini secara remote tanpa perlu mencolok monitor/keyboard terus-menerus.

7

Proses Finish & Reboot

Tunggu hingga proses instalasi mendownload update dan selesai. Setelah muncul tombol "Reboot Now", tekan Enter. Cabut flashdisk dari port USB ketika layar komputer Anda mulai mati atau muncul instruksi pencabutan media boot.

Tip Laptop sebagai Server

Apabila Anda menginstal di laptop fisik, Linux akan otomatis sleep saat layar ditutup. Anda dapat menonaktifkan fitur sleep penutup layar ini setelah masuk ke sistem server nanti.

2. Hubungkan ke Server via SSH

Setelah proses reboot selesai, Mini PC / Laptop Anda kini sudah berfungsi sebagai server (Headless). Letakkan perangkat di dekat router internet, dan kini Anda dapat mencabut monitor serta keyboard fisik di perangkat tersebut.

Cara Mengetahui IP Address Server

Apabila belum sempat mencatat IP server saat instalasi, nyalakan layar Mini PC, login, kemudian ketik perintah berikut pada shell hitam lokal:

Shell Lokal Server 
hostname -I

Hubungkan dari PC/Laptop Utama

Kembali ke PC/Laptop utama Anda yang terhubung di jaringan Wi-Fi/LAN yang sama:

A

Menggunakan PowerShell / Command Prompt (Windows/Mac)

Buka terminal di komputer Anda, lalu jalankan perintah berikut:

Command Prompt / PowerShell 
ssh masandigital@IP_SERVER_LINUX
# Contoh: ssh masandigital@192.168.1.15

Masukkan password user masandigital yang telah Anda buat pada langkah sebelumnya.

B

Menggunakan Aplikasi GUI (PuTTY)

Jika Anda lebih menyukai aplikasi berbasis UI, unduh dan instal PuTTY:

Unduh PuTTY Client Resmi
  1. Buka program PuTTY.
  2. Isi kolom Host Name (or IP address) dengan IP Server Anda.
  3. Isi Port dengan nilai 22.
  4. Klik tombol Open di bagian bawah.
  5. Ketika muncul pop-up peringatan keamanan pertama kali, klik Yes.
  6. Login menggunakan username masandigital dan masukkan password Anda.

3. Instalasi Docker & Docker Compose

Setelah berhasil login via SSH, kita perlu memasang Docker Engine dan Docker Compose Plugin. Ini adalah fondasi utama untuk menjalankan kontainer aplikasi dashboard.

Jalankan perintah-perintah berikut secara berurutan di dalam sesi SSH server Anda:

1

Update Indeks Paket Server

SSH Terminal 
sudo apt-get update && sudo apt-get upgrade -y
2

Pasang Paket Dependensi Awal

SSH Terminal 
sudo apt-get install -y ca-certificates curl gnupg lsb-release
3

Tambahkan Docker Official GPG Key

SSH Terminal 
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
4

Konfigurasikan Repositori Docker

SSH Terminal 
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
5

Pasang Docker Engine & Docker Compose Plugin

SSH Terminal 
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
6

Izinkan User Anda Menjalankan Docker Tanpa Sudo

Agar Anda tidak perlu menulis kata sudo setiap kali memanggil command docker:

SSH Terminal 
sudo usermod -aG docker $USER
newgrp docker
7

Verifikasi Hasil Instalasi

Jalankan perintah ini untuk memastikan Docker dan Docker Compose berjalan dengan benar:

SSH Terminal 
docker --version
docker compose version

4. Kloning & Jalankan Dashboard

Setelah Docker siap, saatnya memasang MasanDigital Dashboard menggunakan berkas installer yang sudah dikonfigurasi.

SSH Terminal 
# 1. Kembali ke folder home user
cd ~

# 2. Kloning repositori GitHub Anda
git clone https://github.com/mychiara/web-server.git masandigital_dashboard

# 3. Masuk ke folder proyek
cd masandigital_dashboard

# 4. Berikan izin eksekusi berkas installer otomatis
chmod +x install.sh

# 5. Jalankan installer sebagai root
sudo bash install.sh
Proses Selesai!

Tunggu proses download base image Python dan download library selesai sampai terminal Anda memunculkan pesan instalasi sukses berwarna hijau.

5. Konfigurasi Awal (Setup Wizard)

Setelah kontainer sukses berjalan di server Anda:

  1. Buka web browser di PC/Laptop/HP utama Anda yang tersambung di jaringan Wi-Fi router yang sama.
  2. Ketik alamat IP Server Anda pada address bar, contoh: http://192.168.1.15.
  3. Anda akan diarahkan ke halaman **Setup Wizard** (karena file keamanan kosong).
  4. Ketik dan buat **Password Admin** baru Anda, lalu konfirmasikan sandi tersebut.
  5. Gunakan sandi tersebut untuk masuk ke dashboard utama. Konfigurasi selesai!

6. Pengelolaan Server & Perintah Penting

Berikut adalah list perintah dasar di terminal SSH yang berguna saat mengoperasikan server harian Anda:

Mengontrol Kontainer Dashboard

SSH Terminal 
# Masuk ke direktori dashboard
cd ~/masandigital_dashboard

# Menghentikan layanan dashboard
docker compose down

# Menyalakan kembali dashboard
docker compose up -d

# Melihat logs backend server dashboard secara real-time
docker compose logs -f

Konfigurasi Tambahan Laptop (Lid Close Settings)

Jika server Anda menggunakan **Laptop fisik**, secara default layar ditutup akan membuat laptop mati (sleep). Matikan fitur ini dengan perintah:

SSH Terminal 
# Buka konfigurasi systemd login
sudo nano /etc/systemd/logind.conf

Cari baris berikut, hilangkan tanda pagar # di depannya dan ubah nilainya menjadi ignore:

HandleLidSwitch=ignore
HandleLidSwitchExternalPower=ignore
HandleLidSwitchDocked=ignore

Simpan (Ctrl+X, ketik Y, Enter) dan terapkan perubahan dengan merestart service:

SSH Terminal 
sudo systemctl restart systemd-logind

FAQ & Solusi Kendala

Berikut adalah beberapa daftar kendala yang sering ditemui oleh pemula:

Pastikan IP server yang Anda masukkan sudah benar dan perangkat Anda terhubung ke router yang sama. Bila Anda menggunakan OS Ubuntu, bisa jadi port 80 diblokir oleh firewall bawaan. Aktifkan port tersebut:

sudo ufw allow 80/tcp
sudo ufw reload

Karena Anda menginstall menggunakan Git Clone, Anda cukup menarik update terbaru dari GitHub dan melakukan rebuild kontainer:

cd ~/masandigital_dashboard
git pull origin main
docker compose up -d --build

1. Masuk ke folder cd ~/masandigital_dashboard
2. Jalankan perintah nano docker-compose.yml.
3. Cari bagian ports: dan ubah baris - "80:5000" menjadi - "8080:5000" (sesuaikan 8080 dengan port pilihan Anda).
4. Simpan file (Ctrl+X, Y, Enter) dan terapkan: docker compose up -d.
